# HPC Job Submission Guide

This directory contains templates and utilities for submitting PRL training jobs to the HPC using IBM LSF.

## Files

- **setup_env_hpc.sh** — Load modules and set environment variables for HPC execution
- **submit_single.sh** — Template for submitting a single training run via `bsub`
- **README_HPC.md** — This file

## Workflow

### Prerequisites

Before running jobs on HPC, ensure:

1. **Environment variables are set** (typically in your `.bashrc` or `.zshrc`):
   ```bash
   export PRL_PROJECT_ROOT="/path/to/prl_project"
   export PRL_DATA_ROOT="/path/to/data"
   export PRL_TRAIN_ROOT="/path/to/training"
   ```

2. **Modules are available** — Update `setup_env_hpc.sh` to load the necessary modules:
   - FSL (for fslroi, fslmaths, fslstats)
   - C3D (for c3d image operations)
   - Python 3.x with MONAI and dependencies
   - CUDA (if using GPU acceleration)

### Single Job Submission

To submit a single training run:

```bash
bash hpc/submit_single.sh --run-dir /path/to/run3
```

This creates a temporary job script and submits it via `bsub`.

### Job Array Submission (HPO)

For hyperparameter optimization, `launch_experiments.py` auto-generates an array submission script:

```bash
# First, generate the experiments
python scripts/generate_experiments.py training/roi_train2/experiment_config.json

# Then, submit the array job
python scripts/launch_experiments.py training/roi_train2/experiment_config.json --hpc
```

This will:
1. Create a `submit_array.sh` script in the `training_work_home` directory
2. Submit the job array via `bsub -J "prl_hpo[1-N]"` where N is the number of runs
3. Each array element processes one HPO configuration in parallel

### Monitoring Jobs

View submitted jobs:
```bash
bjobs
```

View detailed job info:
```bash
bjobs -l <job_id>
```

Kill a job:
```bash
bkill <job_id>
```

Kill all array jobs:
```bash
bkill 0
```

### Output Logs

Logs are saved to `{training_work_home}/logs/`:
- `run_<job_id>_<array_index>.log` — stdout
- `run_<job_id>_<array_index>.err` — stderr

## Customization

Edit `hpc/setup_env_hpc.sh` to match your HPC environment:
- Uncomment and adjust module load commands
- Update resource requests (`#BSUB` directives) in generated job scripts

For `submit_array.sh` (generated by `launch_experiments.py`), customize these `#BSUB` directives:

```bash
#BSUB -n 1                    # Number of CPUs
#BSUB -R "select[gpu]"       # Select GPU nodes
#BSUB -q gpu                  # Queue name (adjust to your HPC)
```

## Troubleshooting

**Module not found:**
- Check available modules: `module avail`
- Adjust module names in `setup_env_hpc.sh`

**Job script syntax error:**
- Verify `PRL_PROJECT_ROOT` and other env vars are set
- Check that paths exist and are accessible on HPC

**Jobs fail immediately:**
- Check logs in `logs/` directory
- Ensure FSL, C3D, and other dependencies are properly loaded
